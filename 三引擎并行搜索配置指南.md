# 🚀 三引擎并行搜索配置指南

## 功能说明

现在系统支持**同时调用三个搜索引擎**进行并行搜索，然后合并结果：
- ✅ **Tavily**：专为 AI 优化
- ✅ **Serper**：Google 搜索 API
- ✅ **百度千帆**：百度搜索 API

### 工作流程

```
用户查询
  ↓
Planner 规划子任务
  ↓
Executor 执行工具调用
  ↓
┌─────────────┬─────────────┬─────────────┐
│   Tavily    │   Serper    │   Qianfan   │  (并行搜索)
└─────────────┴─────────────┴─────────────┘
  ↓             ↓             ↓
合并结果（去重、排序）
  ↓
传递给 DeepSeek Reasoner
  ↓
生成综合答案
```

## 📝 配置步骤

### 步骤 1：获取 Tavily API Key

#### 1.1 注册账号
访问：https://tavily.com/

点击 "Sign Up" 注册（支持 Google/GitHub 登录）

#### 1.2 获取 API Key
登录后在 Dashboard 页面获取 API Key

格式：`tvly-xxxxxxxxxxxxxxxxxxxxxxxx`

#### 1.3 配置到项目
编辑 `backend/.env` 文件：
```bash
TAVILY_API_KEY=tvly-你的实际key
```

**免费额度**：1000 次/月

## ✅ 完整配置示例

编辑 `backend/.env` 文件，应该包含以下内容：

```bash
# DeepSeek API 配置（必需）
DEEPSEEK_API_KEY=sk-你的deepseek_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-reasoner

# 搜索引擎 API 配置
# Tavily - 专为 AI 优化的搜索引擎
TAVILY_API_KEY=tvly-你的tavily_key

# Serper - Google 搜索 API
SERPER_API_KEY=你的serper_key

# 百度千帆搜索
QIANFAN_API_KEY=你的qianfan_key
```

**重要提示**：
- 将 `your_xxx_api_key_here` 替换为实际的 API Key
- 不要在 API Key 两边加引号
- 确保没有多余的空格

### ❌ 配置失败的情况

#### 情况 1：API Key 未配置
```
⚠ Tavily API Key 未配置，跳过
⚠ Serper API Key 未配置，跳过
⚠ Qianfan API Key 未配置，跳过
⚠ 没有可用的搜索引擎，返回模拟数据
```

**解决方案**：检查 `.env` 文件，确保已填入实际的 API Key

#### 情况 2：API Key 无效
```
→ 启动 Tavily 搜索任务...
✗ Tavily 搜索失败: Tavily API error: 401 - Unauthorized
```

**解决方案**：
- 检查 API Key 是否正确
- 确认 API Key 没有过期
- 重新生成 API Key

#### 情况 3：网络问题
```
→ 启动 Tavily 搜索任务...
✗ Tavily 搜索失败: Tavily search timeout
```

**解决方案**：
- 检查网络连接
- 尝试使用代理
- 稍后重试

## 🎯 并行搜索的优势

### 1. 更全面的结果
- 合并三个搜索引擎的结果
- 覆盖更多信息源
- 去重后提供最相关的内容

### 2. 更高的可靠性
- 单个引擎失败不影响整体
- 至少有一个引擎成功即可
- 自动降级处理

### 3. 更快的速度
- 并行执行，不是顺序执行
- 总耗时 ≈ 最慢引擎的耗时
- 通常 2-4 秒完成

### 4. 更好的答案质量
- 多个来源的直接答案
- DeepSeek 可以综合多方信息
- 提供更准确、全面的回答

## 🔧 高级配置

### 切换为顺序搜索

如果你想使用顺序搜索（只用第一个可用引擎），可以修改 `tool_implementations.py`：

```python
# 找到这一行
result = await unified_search.search(
    query=query,
    max_results=max_results,
    search_depth=search_depth,
    parallel=True,  # 改为 False
    **kwargs
)
```

### 调整搜索深度

在 Planner 的提示词中，可以指定搜索深度：

- `basic`：快速搜索（默认）
- `advanced`：深度搜索（更详细，但更慢）

## 🐛 常见问题

### Q1: 为什么只有一个引擎在工作？

**A**: 检查其他引擎的 API Key 是否正确配置。日志会显示：
```
⚠ Serper API Key 未配置，跳过
```

### Q2: 搜索很慢怎么办？

**A**: 
1. 检查网络连接
2. 考虑只配置一个引擎
3. 调整超时时间（在 `search_tools.py` 中）

### Q3: 结果重复怎么办？

**A**: 系统会自动去重（基于 URL），如果仍有重复，可能是不同引擎返回了相同内容但 URL 不同。

### Q4: 如何查看每个引擎返回了什么？

**A**: 查看后端日志的"搜索结果详情"部分，每条结果都标注了来源引擎：
```
[1] [Tavily] 标题...
[2] [Serper] 标题...
```
