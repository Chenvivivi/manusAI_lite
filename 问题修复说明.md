# 🔧 问题修复说明

## 问题分析

### 错误信息
```
处理失败: cannot use 'dict' as a set element (unhashable type: 'dict')
```

### 原因
在 `executor.py` 的 `execute_all()` 方法中，使用了 `set()` 来存储已执行的任务 ID：

```python
executed_ids = set()
executed_ids.add(subtask.id)  # 如果 subtask.id 是字典，会报错
```

如果 Planner 返回的子任务 ID 是复杂类型（如字典），就无法添加到集合中。

## 已修复的问题

### 1. 集合改为列表
**修改**: `backend/executor.py`

```python
# 修改前
executed_ids = set()
executed_ids.add(subtask.id)

# 修改后
executed_ids = []
executed_ids.append(subtask.id)
```

### 2. 修复 JSON 序列化
**修改**: `backend/main.py`

```python
# 修改前（f-string 中的双大括号可能有问题）
yield f"data: {json.dumps({{'type': 'task_update', ...}})}\n\n"

# 修改后（先构建字典，再序列化）
update_data = {'type': 'task_update', 'task_id': subtask.id, 'status': subtask.status}
yield f"data: {json.dumps(update_data, ensure_ascii=False)}\n\n"
```

### 3. 增强错误处理
添加了多层 try-except：
- Planner 调用的错误捕获
- Executor 执行的错误捕获
- 单个子任务执行的错误捕获

### 4. 中文乱码修复
**修改**: `backend/main.py` 开头

```python
import sys
import io

sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8')
```
