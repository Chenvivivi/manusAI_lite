# DeepSeek API 配置说明

## 获取 API Key

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/)
2. 注册并登录账号
3. 进入 API Keys 页面
4. 创建新的 API Key
5. 复制 API Key（注意保密，不要泄露）

## 配置步骤

### 方法一：修改 .env 文件（推荐）

编辑 `backend/.env` 文件：

```env
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-reasoner
```

### 方法二：设置环境变量

Windows PowerShell:
```powershell
$env:DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxx"
```

Windows CMD:
```cmd
set DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

Linux/Mac:
```bash
export DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

## 验证配置

启动后端服务后，如果看到以下警告：

```
!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
警告: 未配置 DEEPSEEK_API_KEY
请在 backend/.env 文件中设置你的 API Key
!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
```

说明 API Key 未正确配置，请检查 `.env` 文件。

## DeepSeek Reasoner 模型说明

- **模型名称**: `deepseek-reasoner`
- **特点**: 支持推理模式，会输出思维链内容
- **max_tokens**: 默认 32K，最大 64K
- **输出内容**:
  - `content`: 最终回答
  - `reasoning_content`: 思维链（推理过程）

## 对话历史管理

- 每个任务（taskId）维护独立的对话历史
- 自动拼接上下文，支持多轮对话
- 只保留 `content`，不保留 `reasoning_content`（符合 API 规范）

## 费用说明

DeepSeek API 按 token 计费，具体价格请查看官方文档。建议：
- 设置合理的 `max_tokens` 限制
- 定期清理不需要的对话历史
- 监控 API 使用量

## 故障排查

### 问题 1: API Key 无效
- 检查 `.env` 文件中的 API Key 是否正确
- 确认 API Key 没有过期
- 检查账户余额是否充足

### 问题 2: 连接超时
- 检查网络连接
- 确认可以访问 https://api.deepseek.com
- 尝试使用代理（如需要）

### 问题 3: 返回 400 错误
- 检查是否在 messages 中传入了 `reasoning_content`
- 确认请求格式符合 API 规范
