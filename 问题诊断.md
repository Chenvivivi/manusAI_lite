# 🔍 "规划中..." 一直显示的问题诊断

## 问题描述

前端发送消息后，一直显示"规划中 • • •"，不刷新页面就看不到回答和思考过程。

## 可能的原因

### 1. 流式数据没有正确接收
前端的 `ReadableStream` 可能没有正确处理 SSE 数据。

### 2. Vue 响应式更新问题
修改对象属性时，Vue 可能没有检测到变化。

### 3. 后端数据没有发送
后端的 `generate()` 函数可能没有正确执行或发送数据。

## 已应用的修复

### 修复 1: 改进 Vue 响应式更新
```javascript
// 使用 getMessage() 函数每次获取最新的消息对象
const getMessage = () => {
    return task.messages.find(m => m.id === loadingMessageId);
};

// 更新子任务状态时，创建新对象而不是直接修改
currentMessage.subtasks[subtaskIndex] = {
    ...currentMessage.subtasks[subtaskIndex],
    status: data.status
};
```

### 修复 2: 添加错误捕获
```javascript
try {
    const data = JSON.parse(line.slice(6));
    // 处理数据...
} catch (parseError) {
    console.error('解析 SSE 数据失败:', parseError, line);
}
```

### 修复 3: 后端添加日志
```python
async def generate():
    print("开始生成流式响应...")
    sys.stdout.flush()  # 强制刷新输出
    # ...
```

## 🧪 诊断步骤

### 步骤 1: 检查浏览器控制台

1. 打开 `frontend/index.html`
2. 按 F12 打开开发者工具
3. 切换到 **Console** 标签
4. 发送一条消息
5. 查看是否有错误信息

**预期输出**:
- 没有红色错误
- 可能有 "解析 SSE 数据失败" 的警告

### 步骤 2: 检查网络请求

1. 在开发者工具中切换到 **Network** 标签
2. 发送一条消息
3. 找到 `/api/chat/stream` 请求
4. 点击查看详情

**检查项**:
- Status: 应该是 200 OK
- Type: 应该是 `text/event-stream`
- Response: 查看是否有数据返回

### 步骤 3: 检查后端日志

查看后端控制台，应该看到：

```
开始生成流式响应...

【步骤 1】调用 Planner 规划子任务...

✓ Planner 输出:
{
  "subtasks": [...]
}

==================================================
【步骤 2】Executor 开始执行子任务...
==================================================
...
```

**如果没有看到这些日志**，说明 `generate()` 函数没有执行。

## 🔧 临时解决方案

如果问题仍然存在，可以尝试以下方法：

### 方案 1: 使用轮询代替流式

修改前端，使用定时轮询获取结果：

```javascript
// 发送请求后，每秒查询一次状态
const pollStatus = setInterval(async () => {
    const status = await fetch(`/api/chat/status/${messageId}`);
    // 更新 UI
}, 1000);
```

### 方案 2: 使用 WebSocket

改用 WebSocket 实现实时通信，更可靠。

### 方案 3: 简化为非流式

暂时改回非流式接口，等所有数据准备好后一次性返回。

## 🐛 调试技巧

### 在浏览器控制台手动测试

```javascript
// 测试流式接收
const response = await fetch('http://localhost:8000/api/chat/stream', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({text: '测试', taskId: 1})
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
    const {done, value} = await reader.read();
    if (done) break;
    console.log(decoder.decode(value));
}
```

### 检查 SSE 格式

后端发送的数据格式应该是：
```
data: {"type":"plan","content":{...}}

data: {"type":"task_update","task_id":1,"status":"completed"}

data: {"type":"content","content":"回答内容"}

```

注意：
- 每行以 `data: ` 开头
- 每条消息后有两个换行符 `\n\n`
